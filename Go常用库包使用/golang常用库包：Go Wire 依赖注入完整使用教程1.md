## 什么是 Wire

Wire 是 Google 开发的 Go 语言依赖注入代码生成工具。与运行时依赖注入框架不同，Wire 在编译时生成代码，具有以下优势:

- **编译时安全**: 在编译阶段发现依赖问题
- **无运行时开销**: 生成的是纯 Go 代码
- **易于调试**: 生成的代码可读性强
- **类型安全**: 充分利用 Go 的类型系统

## 快速开始

### 安装

```bash
go install github.com/google/wire/cmd/wire@latest
```

### 第一个示例

**步骤 1: 创建项目结构**

```
myapp/
├── main.go
└── wire.go
```

**步骤 2: 定义依赖 (main.go)**

```go
package main

import "fmt"

// Database 数据库连接
type Database struct {
    DSN string
}

func NewDatabase(dsn string) *Database {
    return &Database{DSN: dsn}
}

// UserRepository 用户仓储
type UserRepository struct {
    DB *Database
}

func NewUserRepository(db *Database) *UserRepository {
    return &UserRepository{DB: db}
}

// UserService 用户服务
type UserService struct {
    Repo *UserRepository
}

func NewUserService(repo *UserRepository) *UserService {
    return &UserService{Repo: repo}
}

func (s *UserService) GetUser(id int) {
    fmt.Printf("Getting user %d from DB: %s\n", id, s.Repo.DB.DSN)
}

func main() {
    service := InitializeUserService("localhost:3306")
    service.GetUser(1)
}
```

**步骤 3: 编写 Wire 配置 (wire.go)**

```go
//go:build wireinject
// +build wireinject

package main

import "github.com/google/wire"

func InitializeUserService(dsn string) *UserService {
    wire.Build(NewDatabase, NewUserRepository, NewUserService)
    return nil
}
```

**步骤 4: 生成代码**

```bash
wire
```

这会生成 `wire_gen.go` 文件:

```go
// Code generated by Wire. DO NOT EDIT.

//go:generate go run github.com/google/wire/cmd/wire
//go:build !wireinject
// +build !wireinject

package main

func InitializeUserService(dsn string) *UserService {
    database := NewDatabase(dsn)
    userRepository := NewUserRepository(database)
    userService := NewUserService(userRepository)
    return userService
}
```

**步骤 5: 运行程序**

```bash
go run .
# 输出: Getting user 1 from DB: localhost:3306
```

## 核心概念

### Provider (提供者)

Provider 是一个返回值的普通函数，Wire 使用它来构建依赖图。

```go
// 简单 Provider
func NewLogger() *Logger {
    return &Logger{}
}

// 带参数的 Provider
func NewDatabase(config *Config) *Database {
    return &Database{Config: config}
}

// 返回多个值的 Provider (第二个必须是 error)
func NewConnection(dsn string) (*Connection, error) {
    conn, err := sql.Open("mysql", dsn)
    if err != nil {
        return nil, err
    }
    return &Connection{conn}, nil
}
```

### Injector (注入器)

Injector 是 Wire 生成代码的入口函数，定义在 `wire.go` 文件中。

```go
//go:build wireinject

package main

import "github.com/google/wire"

// Injector 函数签名
func InitializeApp(config *Config) (*App, error) {
    wire.Build(
        // 在这里列出所有 Providers
    )
    return nil, nil
}
```

### Provider Set (提供者集合)

Provider Set 用于组织相关的 Providers。

```go
// database/providers.go
var DatabaseSet = wire.NewSet(
    NewDatabase,
    NewUserRepository,
    NewPostRepository,
)

// service/providers.go
var ServiceSet = wire.NewSet(
    NewUserService,
    NewPostService,
)

// wire.go
func InitializeApp() *App {
    wire.Build(
        DatabaseSet,
        ServiceSet,
        NewApp,
    )
    return nil
}
```

## 功能介绍

### 1. 基本依赖注入

```go
type Logger struct{}

func NewLogger() *Logger {
    return &Logger{}
}

type Database struct {
    Logger *Logger
}

func NewDatabase(logger *Logger) *Database {
    return &Database{Logger: logger}
}

// wire.go
func InitializeDatabase() *Database {
    wire.Build(NewLogger, NewDatabase)
    return nil
}
```

### 2. 接口绑定

Wire 可以将具体类型绑定到接口。

```go
// 定义接口
type Logger interface {
    Log(msg string)
}

// 实现
type ConsoleLogger struct{}

func (l *ConsoleLogger) Log(msg string) {
    fmt.Println(msg)
}

func NewConsoleLogger() *ConsoleLogger {
    return &ConsoleLogger{}
}

// 使用接口的服务
type Service struct {
    Logger Logger
}

func NewService(logger Logger) *Service {
    return &Service{Logger: logger}
}

// wire.go
func InitializeService() *Service {
    wire.Build(
        NewConsoleLogger,
        wire.Bind(new(Logger), new(*ConsoleLogger)),
        NewService,
    )
    return nil
}
```

### 3. 结构体字段注入

```go
type Config struct {
    Host string
    Port int
}

type Server struct {
    Config Config  // 注意: 不是指针
}

func NewServer(config Config) *Server {
    return &Server{Config: config}
}

// wire.go
func InitializeServer() *Server {
    wire.Build(
        wire.Struct(new(Config), "Host", "Port"),
        NewServer,
    )
    return nil
}

// 或者使用 "*" 表示所有字段
func InitializeServer() *Server {
    wire.Build(
        wire.Struct(new(Config), "*"),
        NewServer,
    )
    return nil
}
```

### 4. 值绑定

```go
// wire.go
func InitializeApp() *App {
    wire.Build(
        wire.Value("localhost:3306"),  // 绑定字符串值
        wire.Value(8080),               // 绑定整数值
        NewDatabase,
        NewApp,
    )
    return nil
}
```

### 5. 接口值绑定

```go
// wire.go
func InitializeService() *Service {
    wire.Build(
        NewConsoleLogger,
        wire.InterfaceValue(new(Logger), &ConsoleLogger{}),
        NewService,
    )
    return nil
}
```

## 深入使用

### 清理函数 (Cleanup Functions)

当 Provider 返回需要清理的资源时，可以使用 cleanup 函数。

```go
// 定义 Provider
func NewDatabase(dsn string) (*Database, func(), error) {
    db, err := sql.Open("mysql", dsn)
    if err != nil {
        return nil, nil, err
    }
    
    cleanup := func() {
        db.Close()
    }
    
    return &Database{db}, cleanup, nil
}

// wire.go
func InitializeApp(dsn string) (*App, func(), error) {
    wire.Build(
        NewDatabase,
        NewApp,
    )
    return nil, nil, nil
}

// main.go
func main() {
    app, cleanup, err := InitializeApp("localhost:3306")
    if err != nil {
        log.Fatal(err)
    }
    defer cleanup()  // 确保清理资源
    
    app.Run()
}
```

### 字段选择器 (Field Selectors)

从结构体中提取特定字段。

```go
type Config struct {
    Database DatabaseConfig
    Server   ServerConfig
}

type DatabaseConfig struct {
    DSN string
}

// wire.go
func InitializeDatabase(config *Config) *Database {
    wire.Build(
        wire.FieldsOf(new(*Config), "Database"),  // 提取 Database 字段
        NewDatabase,
    )
    return nil
}
```

### 结构体提供者 (Struct Providers)

直接构造结构体而不需要构造函数。

```go
type Server struct {
    DB     *Database
    Logger *Logger
    Port   int
}

// wire.go
func InitializeServer(port int) *Server {
    wire.Build(
        NewDatabase,
        NewLogger,
        wire.Struct(new(Server), "*"),  // 自动注入所有字段
    )
    return nil
}
```

指定部分字段:

```go
wire.Build(
    NewDatabase,
    NewLogger,
    wire.Value(8080),
    wire.Struct(new(Server), "DB", "Logger", "Port"),
)
```

## 高级特性

### 1. 条件注入

虽然 Wire 不直接支持条件注入,但可以通过不同的 Injector 实现:

```go
// wire_dev.go
//go:build wireinject && dev

func InitializeApp() *App {
    wire.Build(
        NewDevLogger,
        NewApp,
    )
    return nil
}

// wire_prod.go
//go:build wireinject && prod

func InitializeApp() *App {
    wire.Build(
        NewProdLogger,
        NewApp,
    )
    return nil
}
```

### 2. Provider Sets 组织

```go
// internal/providers/database.go
package providers

var DatabaseProviders = wire.NewSet(
    NewDatabase,
    NewUserRepo,
    NewPostRepo,
)

// internal/providers/service.go
var ServiceProviders = wire.NewSet(
    NewUserService,
    NewPostService,
)

// internal/providers/providers.go
var AllProviders = wire.NewSet(
    DatabaseProviders,
    ServiceProviders,
    NewLogger,
    NewConfig,
)

// cmd/app/wire.go
func InitializeApp() (*App, func(), error) {
    wire.Build(
        providers.AllProviders,
        NewApp,
    )
    return nil, nil, nil
}
```

### 3. 多个 Injector

```go
// wire.go
//go:build wireinject

package main

import "github.com/google/wire"

// HTTP 服务器
func InitializeHTTPServer() (*HTTPServer, error) {
    wire.Build(HTTPServerSet)
    return nil, nil
}

// gRPC 服务器
func InitializeGRPCServer() (*GRPCServer, error) {
    wire.Build(GRPCServerSet)
    return nil, nil
}

// CLI 应用
func InitializeCLI() (*CLI, error) {
    wire.Build(CLISet)
    return nil, nil
}
```

### 4. 泛型支持 (Go 1.18+)

```go
type Repository[T any] struct {
    DB *Database
}

func NewRepository[T any](db *Database) *Repository[T] {
    return &Repository[T]{DB: db}
}

// 使用时需要指定具体类型
type User struct {
    ID   int
    Name string
}

func InitializeUserRepo() *Repository[User] {
    wire.Build(
        NewDatabase,
        NewRepository[User],
    )
    return nil
}
```

## 实战示例

### 示例 1: Web 应用程序

```go
// config.go
package main

type Config struct {
    DatabaseDSN string
    ServerPort  int
    LogLevel    string
}

func NewConfig() *Config {
    return &Config{
        DatabaseDSN: "localhost:3306",
        ServerPort:  8080,
        LogLevel:    "info",
    }
}

// logger.go
type Logger interface {
    Info(msg string)
    Error(msg string)
}

type ZapLogger struct {
    level string
}

func NewZapLogger(config *Config) *ZapLogger {
    return &ZapLogger{level: config.LogLevel}
}

func (l *ZapLogger) Info(msg string) {
    fmt.Printf("[INFO] %s\n", msg)
}

func (l *ZapLogger) Error(msg string) {
    fmt.Printf("[ERROR] %s\n", msg)
}

// database.go
type Database struct {
    DSN    string
    Logger Logger
}

func NewDatabase(config *Config, logger Logger) (*Database, func(), error) {
    db := &Database{
        DSN:    config.DatabaseDSN,
        Logger: logger,
    }
    
    logger.Info("Database connected")
    
    cleanup := func() {
        logger.Info("Database connection closed")
    }
    
    return db, cleanup, nil
}

// repository.go
type UserRepository struct {
    DB     *Database
    Logger Logger
}

func NewUserRepository(db *Database, logger Logger) *UserRepository {
    return &UserRepository{DB: db, Logger: logger}
}

func (r *UserRepository) GetUser(id int) *User {
    r.Logger.Info(fmt.Sprintf("Fetching user %d", id))
    return &User{ID: id, Name: "Test User"}
}

// service.go
type UserService struct {
    Repo   *UserRepository
    Logger Logger
}

func NewUserService(repo *UserRepository, logger Logger) *UserService {
    return &UserService{Repo: repo, Logger: logger}
}

func (s *UserService) GetUser(id int) *User {
    return s.Repo.GetUser(id)
}

// handler.go
type UserHandler struct {
    Service *UserService
    Logger  Logger
}

func NewUserHandler(service *UserService, logger Logger) *UserHandler {
    return &UserHandler{Service: service, Logger: logger}
}

// server.go
type Server struct {
    Handler *UserHandler
    Port    int
    Logger  Logger
}

func NewServer(handler *UserHandler, config *Config, logger Logger) *Server {
    return &Server{
        Handler: handler,
        Port:    config.ServerPort,
        Logger:  logger,
    }
}

func (s *Server) Start() {
    s.Logger.Info(fmt.Sprintf("Server starting on port %d", s.Port))
}

// providers.go
var DatabaseSet = wire.NewSet(
    NewDatabase,
    NewUserRepository,
)

var ServiceSet = wire.NewSet(
    NewUserService,
)

var HandlerSet = wire.NewSet(
    NewUserHandler,
)

// wire.go
//go:build wireinject

package main

import "github.com/google/wire"

func InitializeServer() (*Server, func(), error) {
    wire.Build(
        NewConfig,
        NewZapLogger,
        wire.Bind(new(Logger), new(*ZapLogger)),
        DatabaseSet,
        ServiceSet,
        HandlerSet,
        NewServer,
    )
    return nil, nil, nil
}

// main.go
type User struct {
    ID   int
    Name string
}

func main() {
    server, cleanup, err := InitializeServer()
    if err != nil {
        fmt.Printf("Failed to initialize server: %v\n", err)
        return
    }
    defer cleanup()
    
    server.Start()
}
```

    ProductRepo *repository.ProductRepo
}

func NewProductService(repo *repository.ProductRepo) *ProductService {
    return &ProductService{ProductRepo: repo}

var ServiceSet = wire.NewSet(
    NewUserService,
    NewProductService,

// API 层
package api

type API struct {
    UserService    *service.UserService
    ProductService *service.ProductService
}

func NewAPI(userSvc *service.UserService, productSvc *service.ProductService) *API {
    return &API{
        UserService:    userSvc,
        ProductService: productSvc,
    }
}

import (
    "github.com/google/wire"
    "myapp/api"
    "myapp/database"
    "myapp/repository"
    "myapp/service"

func InitializeAPI(dsn string) (*api.API, error) {
    wire.Build(
        database.PostgresSet,
        repository.RepositorySet,
        service.ServiceSet,
        api.NewAPI,
    )
    return nil, nil
}

### 1. 文件组织

### 2. Provider 命名约定

- 使用 `New` 前缀: `NewDatabase`, `NewUserService`
- Provider Set 使用描述性名称: `DatabaseSet`, `ServiceSet`
- Injector 使用 `Initialize` 前缀: `InitializeApp`, `InitializeServer`

### 3. 错误处理

```go
// 推荐: 返回 error
func NewDatabase(dsn string) (*Database, error) {
    if dsn == "" {
        return nil, errors.New("dsn is required")
    }
    return &Database{DSN: dsn}, nil
}

// 不推荐: panic
func NewDatabase(dsn string) *Database {
    if dsn == "" {
        panic("dsn is required")
    }
    return &Database{DSN: dsn}
}
```

### 4. 使用接口

```go
// 定义接口
type Logger interface {

type Storage interface {

// 在 Provider 中使用接口

    return &Service{
        Logger:  logger,
        Storage: storage,
    }
}
```

## 常见问题

### Q1: Wire 生成的代码可以手动修改吗?

**不可以**。`wire_gen.go` 是自动生成的,每次运行 `wire` 命令都会覆盖。如果需要自定义逻辑,应该在 Provider 函数中实现。

### Q2: 如何处理循环依赖?

Wire 会在编译时检测循环依赖并报错。解决方法:
- 重新设计依赖关系
- 使用接口打破循环
- 将共同依赖提取到单独的组件

```go
// 错误: A -> B -> C -> A

// 正确: 使用接口
type EventBus interface {
    Publish(event Event)
}

type ServiceA struct {
    Bus EventBus  // 使用接口而不是具体类型
}
```

### Q3: Wire 支持可选依赖吗?

不直接支持。但可以通过以下方式实现:

```go
// 方式 1: 使用指针
func NewService(logger *Logger) *Service {
    if logger == nil {
        logger = NewDefaultLogger()
    }
    return &Service{Logger: logger}
}

// 方式 2: 使用不同的 Provider Set
var WithCacheSet = wire.NewSet(NewCache, NewServiceWithCache)
var WithoutCacheSet = wire.NewSet(NewServiceWithoutCache)
```

### Q4: 如何在 Wire 中使用单例?

```go
var (
    loggerInstance *Logger
    once           sync.Once
)

func NewLogger() *Logger {
    once.Do(func() {
        loggerInstance = &Logger{}
    })
    return loggerInstance
}
```

### Q5: Wire 和其他 DI 框架的区别?

| 特性 | Wire | Uber Fx | Go-inject |
|------|------|---------|-----------|
| 类型 | 编译时代码生成 | 运行时反射 | 运行时反射 |
| 性能 | 最快(无运行时开销) | 较慢 | 较慢 |
| 调试 | 容易(生成纯 Go 代码) | 困难 | 困难 |
| 类型安全 | 编译时检查 | 运行时检查 | 运行时检查 |
| 学习曲线 | 平缓 | 陡峭 | 中等 |

### Q6: 如何调试 Wire 生成的代码?

1. 查看 `wire_gen.go` 文件
2. 使用 `wire` 命令的 `-v` 选项查看详细信息
3. 检查 Provider 的参数和返回值类型是否匹配

### Q7: Wire 支持泛型吗?

支持 (Go 1.18+),但需要在使用时指定具体类型:

```go
func NewRepository[T any](db *Database) *Repository[T] {
    return &Repository[T]{DB: db}
}

// wire.go - 必须指定具体类型
func InitializeUserRepo() *Repository[User] {
    wire.Build(NewDatabase, NewRepository[User])
    return nil
}
```

### Q8: 如何组织大型项目的 Providers?

```go
// 按层级组织
var DataLayer = wire.NewSet(DatabaseSet, RepositorySet)
var ServiceLayer = wire.NewSet(ServiceSet)
var APILayer = wire.NewSet(HandlerSet)

// 按功能模块组织
var UserModule = wire.NewSet(
    NewUserRepo,
    NewUserService,
    NewUserHandler,
)

var ProductModule = wire.NewSet(
    NewProductRepo,
    NewProductService,
    NewProductHandler,
)

// 组合使用
func InitializeApp() *App {
    wire.Build(
        CommonSet,
        UserModule,
        ProductModule,
        NewApp,
    )
    return nil
}
```

## 总结

Wire 是一个强大且高效的依赖注入工具,通过编译时代码生成避免了运行时开销。主要优势:

- **编译时安全** - 在编译阶段发现问题 
- **零运行时开销** - 生成纯 Go 代码 
- **易于调试** - 生成的代码清晰可读 
- **类型安全** - 充分利用 Go 类型系统 
- **无需反射** - 性能优异 

开始使用 Wire,让你的 Go 项目更加模块化和可维护!
